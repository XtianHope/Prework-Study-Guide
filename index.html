HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for the webpage. This element is required and crucial for SEO.
  • SEO : Search Engine Optimization, is the process of improving the ranking of a website in search engines such as Google or Bing.
  • The title tag is always displayed inside of the head.
  • An anchor creates links to the same webpage or other webpages. The anchor element is incomplete without an href attribute.
  • The href attribute is a hyperlink reference.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The metal element defines metadata, such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • Element refers to the entire element from opening to closing tag.
  • Tag refers only to what appears inside the angled brackets.
  • Child elements are elements that are nested inside other elements.
  • Parent elements is known as the body element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Every HTML element is like a box, and is sometimes referred to as the "CSS Box Model".
  • Inline CSS is used when a developer wants to style an element directly in the HTML file.
  • Internal CSS contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page.
  • External CSS is the most common way to style a webpage. It is preferred because it allows the developer to keep all their CSS rules in a seperate file, making changes to design easier.
  • Seperation of concerns is seperating code into different files based on how they will be used. "Each section of code should have its own responsibility".
  • A selector defines the elements or attributes to which the rules/declarations will apply.
  • The rel attribute specifies the relationship between the current document and linked documents.
  • Declaration contains two important components : the CSS property we want to apply, and the value of the property.
  • DRY code is used to greatly reduce the overall size and complexity of a code base by grouping two or more elements that will share the same styling. This avoids repition and more lines of code.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • "*" Is a selector, also known as a wild card symbol, that applies a rule to all the elements visible on the page.
  • CSS are not automatically recognized by VS just because it resides in the same repository. We must use an HTML link element to connect them.
  • Block assigns each image to start on a new line.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • pwd: Lets us know the full pathname of the current working directory.
  • cd ..: Takes us back one step/location.
  • mkdir: Will create a new folder(s).
  • mv: Moves a file.
  • cd : Navigates us to a folder or file that we identify directly after the command.
  • ls: Displays a list of files within the working folder/location.
  • code .:Opens VS code.
  • git add -A: Stages changes that were made.
  • git commit -m "insert change notes here": Commits the changes that were made.
  • git pull origin main: Navigates us to the main branch.
  • git push origin "insert branch name here": Pushes up all of our changes.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array is used to store/hold a group of related data.
  • Control flow is the order in which a computer executes code in a script.
  • A topics variable contains a string of topics.
  • A string is always surrounded by quotation marks.
  • In JS we use the "var" keyword to give a variable a name. We also use an assignment operator "=" to assign the speicif cata that we want stored in our variable.
  • In JS code, the lines of code end with a semi-colon.
  • A function is a set of instructions that tells the computer how to perform a certain task. These do not automatically execute, you must call them. A functions name is always followed by parenthesis
  • ""===" Is a known as Strict Equality Operator. This checks to see if two values are equal, and returns a boolean result (true) if the values are equal or (false) if they are not equal.
  • Commenting out code is a way to make a portion of code inactive without deleting it. You can do this by "/*" before the code and "*/" after the code you would like to comment out.
